## 🎨 Visual Reference (IMPORTANT)

If Figma screenshots or images are provided:
- Treat them as PRIMARY design source
- Match layout, spacing, and hierarchy based on visual structure
- Do NOT rely only on codebase aesthetics

## 🧩 Component Mapping Rules

When refactoring UI:

- If visual pattern resembles a card → reuse Card component
- If pattern resembles CTA → reuse Button component
- If layout is repeated → extract reusable section pattern

If uncertain:
- Prefer reuse over creation

## 🎯 Visual Matching Criteria

The result must match reference design in:

- Layout structure
- Spacing rhythm
- Typography hierarchy
- Visual weight distribution

Pixel-perfect is NOT required.
Structural similarity is REQUIRED.

## Style Consistency Rule

All sections must visually belong to the same system.

If a new UI pattern is introduced, it must be reused across at least 2 sections.
